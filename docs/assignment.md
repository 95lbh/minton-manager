# 코트 자동 배정 알고리즘

> 상태: 확정 v1 · 2026-05-30 · 구현: `src/server/services/assignment.ts` (순수 함수, DB 비의존)

## 1. 배정 방식 두 가지
- **수동 배정**: 관리자가 직접 멤버를 코트에 지정. 알고리즘은 검증만(출석/중복/인원).
- **자동 배정**: 알고리즘이 추천. 아래 룰 적용. 추천일 뿐 관리자가 조정 가능.

## 2. 게임 인원
- 기본 4명(복식 2:2), 설정으로 2명(단식 1:1) 허용. `gameSize: 2 | 4`.

## 3. 자동 배정 팀 구성 모드 (`composition`)
- `free` 성별 무관 / `mens` 남복 / `womens` 여복 / `mixed` 혼복(각 팀 1남1녀).
- 모드에 따라 후보 풀을 성별로 먼저 거른다. `mixed`는 남/녀 수가 부족하면 구성 불가(사유 반환).

## 4. 우선순위 (상위가 강함) — **공정성 최우선**
1. **하드 제약**: 출석 O · 현재 게임중/휴식중 X · 정확히 gameSize명 · (mixed면 성비) · 수동 잠금 포함.
2. **공정성**: 게임 수 적은 사람 → 오래 쉰 사람 우선. **가장 오래 기다린 사람은 반드시 포함(anchor)**. → 후보 선정 단계에서 보장.
3. **파트너 중복 회피** (소프트, 가장 강함)
4. **상대 중복 회피** (소프트, 파트너보다 약함)
5. **실력 균형**: 팀 합산 실력차 최소 (+게임 내 편차 약하게)

> 구조: **공정성=후보 선정**, **중복/실력=팀 구성**으로 분리해 충돌 해결.

## 5. 실력 등급
- S/A/B/C/D ↔ 5/4/3/2/1 (`SKILL_VALUE`). 미상=B(3). 팀 실력차 = 팀 합산 차.

## 6. 중복 이력
- **오늘 세션만**. 파트너/상대 횟수 + 마지막으로 만난 게임 순번.
- **최신성 가중**: 직전 게임에서 또 만나면 패널티 가중(`recencyHalfLife` 기준 감쇠).
- 파트너 중복 가중치 > 상대 중복 가중치.

## 7. 점수(cost, 낮을수록 좋음)
```
cost = wPartner · Σ 파트너중복(같은팀쌍) · (1+최신성)
     + wOpponent · Σ 상대중복(교차쌍) · (1+최신성)
     + wSkill · |팀A합 − 팀B합|
     + wSpread · (게임내 최고−최저 실력)
```
프리셋: `balanced`(기본) / `skill`(실력 우선) / `variety`(다양성 우선). 관리자는 숫자 대신 프리셋 선택.

## 8. 알고리즘
1. 후보 = 대기자 → composition 성별 필터.
2. 공정성 정렬 `[게임수 asc, 마지막게임순번 asc, 대기시작 asc]`.
3. anchor(잠금/최장대기/장기미출전) 포함하도록 window = 상위 K명 (K=gameSize+slack).
4. window에서 anchor 포함 gameSize 조합 전수 → 팀 분할(복식 3, 혼복은 1남1녀 분할만) → cost 최소 선택. 상위 3 대안 제공.
5. 유효 조합 없으면 window 확대 → 그래도 없으면(주로 mixed 성비) `null` + 사유.
6. 출력: 추천 조합·팀, **배정 이유**, 경고(완화 발생), 제외자·사유, 대안.

## 9. 멀티 코트
여러 코트 동시 배정은 그리디: 코트별 best 뽑고 해당 인원 제거 후 반복(`recommendRound`).

## 10. 복잡도
K=8·복식 → 3·C(8,4)=210 평가. 무시 가능.
